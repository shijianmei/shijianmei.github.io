rebase和 merge 的区别:

git merge 会生成一个新得合并节点，而rebase不会.

如两个分支:test 和 master

  D---E test

 /

A—B—C—F master

使用merge合并：

  D--------E

 /          \

A—B—C—F—-G test, master

而使用rebase则：

A—B—D—E—C’—F’ test, master

使用git pull时默认是merge， 加 –rebase参数使其使用rebase方式
